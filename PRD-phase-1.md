# 一期 PRD：塔罗占卜 Web

## 1. 项目概述
- **项目目标**：为塔罗爱好者提供一个沉浸式的 AI 塔罗占卜 Web 体验
- **要解决的问题**：传统塔罗占卜需要真人塔罗师，门槛高、即时性差；现有线上占卜工具缺乏仪式感和个性化解读
- **一期要验证的核心价值**：沉浸式占卜仪式 + LLM 个性化解读，能否让用户愿意反复使用并消耗积分

## 2. 目标用户
- **主要用户**：塔罗爱好者，有一定基础认知，追求专业占卜体验
- **次要相关方**：管理员（通过轻量后台管理积分充值）
- **关键使用场景**：用户心里有困惑 → 打开网站 → 沉浸式抽牌 → 获得结构化 AI 解读 → 保存/分享结果

## 3. 需求边界

### 3.1 一期纳入范围
- 用户注册/登录（邮箱或 OAuth）
- 积分系统（抽卡 1 分 + 解读 2 分，新用户赠 100 分）
- 单种牌阵：经典三张牌（过去 / 现在 / 未来）
- 沉浸式占卜仪式：问题输入 → 氛围过渡 → 洗牌动画 → 选三张牌 → 翻牌揭示
- LLM 结构化解读（牌面含义 / 正逆位 / 对问题的影响 / 建议）
- 78 张塔罗牌基础含义库 + DeepSeek API 个性化解读
- 牌面展示（Rider-Waite 公共领域牌图）
- 占卜历史记录（列表 + 详情回顾 + 收藏）
- 结果卡片图生成与分享
- 中/英文界面切换
- 轻量管理后台（用户列表、积分余额、积分充值记录）

### 3.2 一期明确不做
- 其他牌阵（凯尔特十字、单张、自定义牌阵等）
- 对话式多轮占卜
- 在线支付（充值走后台手动操作）
- 社区/公开投稿/他人结果浏览
- 移动端 App 或小程序
- 牌面图 AI 生成
- 完整的统计分析面板
- 多角色权限体系（仅区分普通用户和管理员）
- SEO（纯 Web App，不依赖搜索引擎流量）

## 4. 核心业务流程
1. 用户访问首页 → 注册/登录（获得 100 积分）
2. 点击"开始占卜" → 输入想占卜的问题
3. 进入仪式场景：暗黑氛围界面 → "洗牌"交互动画
4. 从洗好的牌中依次选出 3 张（代表过去/现在/未来）
5. 三张牌依次翻转揭示 → 展示牌面图 + 名称 + 正逆位
6. 调用 DeepSeek API，传入牌面信息 + 用户问题 → 流式输出结构化解读
7. 解读完成 → 展示完整结果 → 用户可收藏、生成分享卡片、查看历史
8. 积分扣减：抽牌 -1，解读 -2（共 -3）

## 5. 一期功能清单

### 5.1 必做功能

| 功能项 | 目标 | 验收要点 |
|--------|------|----------|
| 用户注册/登录 | 身份识别 + 积分关联 | NextAuth.js + OAuth/邮箱；注册成功赠送 100 积分 |
| 积分系统 | 控制 LLM 调用成本 | 抽卡扣 1 分、解读扣 2 分；余额不足时提示且阻止操作；积分变动记录可查 |
| 78 张塔罗牌基础库 | 牌面数据源 | 包含牌名（中/英）、正逆位关键词、基础含义描述；以 JSON/DB 形式存储 |
| 占卜仪式流程 | 核心体验差异化 | 问题输入页 → 氛围切换 → 洗牌动画 → 抽 3 张牌 → 翻转揭示，动画流畅不卡顿 |
| DeepSeek API 集成 | 个性化解读 | 将牌面信息 + 用户问题构造成 prompt → 流式返回结构化解读；处理超时/报错 |
| 解读结果页 | 展示解读内容 | 结构化分区：牌面含义、正逆位说明、对问题的影响、综合建议；支持流式渲染 |
| 历史记录 | 用户回看 | 列表展示历史占卜（时间、问题、牌面摘要）；点击进入详情；支持收藏/取消收藏 |
| 结果卡片图生成 | 分享裂变 | 前端截图生成卡片（html2canvas 等），含牌面、问题、金句；可下载/复制 |
| 中英文切换 | 多语言覆盖 | i18n 方案，界面文本 + 牌库数据双语；URL 或设置切换 |
| 轻量管理后台 | 积分管理 | 用户列表、积分余额查看、手动增加积分（充值记录留痕） |

### 5.2 可延后功能

| 功能项 | 延后原因 |
|--------|----------|
| 更多牌阵 | 先验证三张牌闭环，再扩展 |
| 在线支付 | 支付对接复杂，一期手动后台充值足够 |
| 邮件通知 | 一期无强通知场景 |
| 日/周占卜限制 | 积分机制已够做频次控制 |

## 6. 第一期交付成果
- **Web 应用**：部署在 Vercel 上，可公开访问
- **完整占卜闭环**：注册 → 登录 → 占卜仪式 → 解读 → 历史 → 分享 全链路可用
- **管理后台**：管理员可查看用户积分并手动充值
- **源代码仓库**：含 README 部署说明

## 7. 验收标准
- 新用户注册后自动获得 100 积分
- 完成一次完整占卜消耗 3 积分（抽牌 1 + 解读 2），余额不足报错
- 洗牌 + 选牌 + 翻牌动画在主流浏览器 60fps
- DeepSeek 解读流式展示，首字延迟 < 3 秒
- 历史记录可查看、可收藏，收藏状态持久化
- 分享卡片生成清晰美观，信息完整
- 中英文切换后界面文本和牌名正确切换
- 管理后台可查用户积分、手动增减积分，积分变动有记录

## 8. 风险、假设与待确认项
- **风险**：DeepSeek API 偶发超时或返回格式不稳定 → 前端需做重试 + 兜底文案
- **风险**：Vercel 免费额度限制（函数执行时间 10s）→ 流式响应需走 edge 或合理设置超时
- **假设**：Rider-Waite 牌图版权在公共领域，可合法使用
- **假设**：用户接受"先登录才能占卜"的流程，不作为体验障碍
- **假设**：一期不需要邮件验证，注册即可用
- **待确认**：DeepSeek API 的具体模型（deepseek-chat 还是 deepseek-reasoner？）
- **待确认**：是否需要针对敏感问题（健康、法律等）做免责声明或内容过滤
